What this is

The task

One shared 10k BPE tokenizer for 4 languages. For each, fertility = tokens ÷ faithful units, where a unit is one letter/number run or one visible punctuation mark.

The rule that matters

The tokenizer must be faithful: decode(encode(text)) keeps every visible character. A tokenizer that drops punctuation or case is rejected — its ratios don't count.

Our approach

HF BPE + Metaspace (reversible), no normalizer (so it's byte-exact), balanced across languages by per-language training weights to minimise the fertility spread.

Faithfulness — try it

The core requirement: encoding then decoding must return the same visible text. Type anything (punctuation, numbers, brackets, URLs, Indic script) and watch it round-trip.

Reproducibility

A real HuggingFace tokenizer: the grader loads tokenizer.json with Tokenizer.from_file(...) and runs evaluate.py on the shipped corpus/*.faithful.txt. Rebuild it with python build_corpus.py && python train.py, score it with python evaluate.py, or run it without Python via node tokenizer_js.mjs. The JS encoder reproduces the HF token counts exactly and round-trips byte-for-byte.
