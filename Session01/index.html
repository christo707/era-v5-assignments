Part 1 · S1-1

Activations exist for a reason

A model with no nonlinearity can only draw a straight boundary, so it cannot separate two interleaved / concentric rings. Adding a single ReLU hidden layer can. Only the activation changes — the boundary picture is the money shot.

The claim

A purely linear model (a single layer + sigmoid) can only carve space with one straight line. Concentric rings aren't linearly separable, so it's stuck near chance.

The concept

Stacking linear layers is still linear. A nonlinearity like ReLU bends space, letting the network compose many lines into a curved, closed boundary that wraps the inner ring.

What we built

~300 noisy 2D points as two rings (inner = class 0, outer = class 1). Both models train live in your browser; we plot each decision boundary and its accuracy.

Part 2 · S1-2

Depth without nonlinearity is a lie

Five stacked linear layers collapse to a single linear map, so a 5-layer linear net is no stronger than 1 layer — both fail the ring task identically. Insert ReLUs between the same five layers and it suddenly solves it.

The claim

Stacking linear layers buys nothing: W₅W₄W₃W₂W₁ · x is just (W₅W₄W₃W₂W₁) · x — one matrix. So 5 linear layers = 1 linear layer = one straight boundary.

The concept

Depth only adds power when something nonlinear sits between the layers. The ReLUs stop the matrices from merging, so each layer can bend space independently.

What we built

Same ring data, three models: 1 linear layer, 5 linear layers (no activation), and 5 layers + ReLU. Then we literally multiply the five matrices and show the product is one matrix.

Part 3 · S1-3

Embeddings learn similarity from nothing but next-token

Trained only to predict the next token in a tiny synthetic grammar, the embedding table clusters related tokens — even though similarity was never supplied.

The claim

Nobody tells the model "cat is like dog." Yet because same-category tokens share the same next-token distribution, the embedding table sorts them into groups on its own.

The concept

To predict the next token well, tokens that behave the same must produce the same output — so the optimizer pushes their embeddings together. Meaning emerges from prediction. This is why word/token embeddings work at all.

What we built

A toy language — animals cat dog cow, verbs eat chase see, fruits apple mango — with category rules. A tiny embed → softmax next-token model, then a 2D PCA plot of the learned 16-D embeddings.
