Cleaning the Indic Corpus & Geospatial Data

The session lays out a cleaning pipeline as an ordered sequence of stages. I counted them, picked two datasets from our Session-3 catalog — an Indic web corpus and a geospatial gazetteer — ran all the cleanups for real, and report the honest statistics below.

1How many strategies — and what they are

The definitive list is stated in §14: normalization, format discipline, quality filtering, deduplication, language validation, PII removal, decontamination, and the manifest. That is 8 cleaning strategies (9 pipeline stages if you count extraction, inherited from Session 3 as stage 0). They are an ordered sequence — each depends on the one before — and one rule sits under all of them: cleaning happens before the content hash is computed.

5Other strategies & concerns cleaned up

The filter-bias trap (the headline finding). A generic English-tuned quality filter (Gopher-style stop-word rule) would have wrongly discarded — of — (—) genuinely good Indic documents — because Indic text has no English stop-words. Our script-aware filter keeps them. Filtering is not neutral cleaning; it decides which languages the model can speak.

  • Brahmic joiner preservation — normalization strips ZWSP/BOM/bidi overrides but keeps ZWJ/ZWNJ (real linguistic content in Indic scripts). A generic "strip all invisibles" cleaner mangles Indic while believing it helps.
  • Indic-PII precision/recall — regex identifiers mask cleanly; a name-detector over-masks common Indic names/place-names, so name masking must trade recall for precision on Indic (kept conservative here).
  • Global vs local dedup — near-dup dedup must run over the whole corpus on one large-memory machine; per-shard dedup leaves cross-shard copies. (Text sample already deduped upstream → 0 found; geo showed real dupes.)
  • Cartographic sovereignty — — geo entries share a claim with PK/CN and must follow the India-official (Survey of India) stance, not the viewer-geography "disputed" framing Western sources use.
  • Indic-name coverage gap — only — of India's places carry a native-script name in GeoNames: a real India-first data deficiency that enrichment (LGD/Indic gazetteers) must fix.
  • Decontamination + canaries — near-dup leakage scan against a held-out set (— leaks in this clean sample) and a planted canary string that is detectable after the fact.
  • Manifest & determinism — every shard gets a content-hash manifest (source, license, cleaning-script hash, token count, language breakdown); the same input reproduces the same hash.
