Anvesha: A 40B India-First LLM

अन्वेषा · "the quest" — inquiry & exploration

A dense 40B model at ~Gemma-4 quality, built to lead on coding, agentic/tool-use, Indic languages, and an India-first worldview — a model whose defaults are Indian (law, geography, currency, language) yet adapts abroad. Below: the decisions, not the prose.

40B dense12T tokens (~300:1) 200,704 vocab128K context~2.9e24 FLOPs (~2M H100-h)
Objectives tagged throughout: CODEAGENTICINDICINDIA-FIRST

★Why "India-first" — engineered, not asserted

India-first = the model's defaults are Indian (adapts when a prompt is explicitly elsewhere). It's manufactured from Indian data at every layer:

1Model & scale

Dense for cost-sensitive single-GPU serving (~24 GB @4-bit, easy LoRA); over-trained for a permanently cheaper, stronger model.

Dense 40B (active=total); MoE kept as a future high-throughput variant.
12T tokens ≈300:1 (Chinchilla ≈0.8T; peers Llama-3-70B 15T, Gemma-3-27B 14T).
128K context — train 32K → RoPE 10k→1M (global), local:global 5:1.
~2.88e24 FLOPs (6ND) ≈ 2M H100-h ≈ 37 days on 2,000 H100s.

3Post-training & alignment

SFT.
  • Tulu-3 (939K) + OpenHermes-2.5 (~1M) general.
  • CODEAGENTIC function-calling schemas, repo-edit + ReAct traces, long-context repo SFT.
  • INDIC native + IndicTrans2-translated instructions + Hinglish.
  • INDIA-FIRST Indian law / civics / culture Q&A.
Preference / RL.
  • DPO (UltraFeedback / Tulu-3 + on-policy) → RLVR (GRPO) with verifiable rewards: code unit-tests, math answer-check, IFEval constraints.
Safety.
  • IndicAlign-Toxic (123K) + Hinglish jailbreak-refusal; test Indic safety directly; track over-refusal separately.

4Cleaning — one ordered pipeline

Cheap ops first, classifiers after heuristics, decontamination last. Indic-specific steps in green.

  1. Extract (trafilatura) → URL blocklist.
  2. Language-ID + route — fastText (English) + IndicLID/GlotLID for 22 Indic langs incl. romanized.
  3. Indic Unicode/script normalization before dedup (nukta/matra/zero-width) so variants collapse; keep code-mixed.
  4. Heuristic quality (C4 + Gopher + FineWeb) with per-language thresholds.
  5. Model quality classifier (FineWeb-Edu ≥3) + Indic-adapted classifier.
  6. PII masking + CODE secret scrub (StarCoder2); toxicity — keep toxic→safe pairs for alignment.
  7. Dedup cascade: exact → MinHash-LSH → SemDeDup; looser Jaccard for scarce Indic.
  8. CODE license filter, minified/auto-gen removal, syntax validity, near-dedup.
  9. Benchmark decontamination last — n-gram + semantic match (catches translated Indic leakage).
  10. INDIA-FIRST curation — source-boost Indian gov/legal/civic/news; FineWeb2 rehydration for low-resource Indic; don't let Western filters strip India-context content.

INDIA-FIRST — measured, not asserted: a counterfactual defaults probe over {₹, lakh/crore, dd-mm-yyyy, IST, BNS/BNSS/BSA, civic entities, Indian names}. Two arms — India-Default Rate (neutral prompts) and Context-Adaptation Rate (prompts set abroad) — both must be high (kills reflexive over-localization). Plus culture sets (BLEnD-India, NormAd, DOSA, DRISHTIKON, MILU), reported worst-case per region/language, gated by rubric + ≥3 expert raters (α ≥0.7).

200,704
byte-level BPE vocab (≈200K, ×256-aligned)
Three forces converge:
  1. 40B scaling-law optimum ≈ 155–180K.
  2. 11+ Indic scripts + code push up.
  3. Empirical plateau ~200K (IndicSuperTokenizer; GPT-4o o200k lands ~200K).
Reject 256K (~0 extra fertility, more softmax) and 128K (3–8× worse Indic).
Cost / settings. Embedding ≈ 4–6% of 40B — cheap vs the Indic tax. Byte fallback, split digits, preserve whitespace/operators, NFC-normalize Indic, shared-script merges, reserve ~2–4K slots.

7Geospatial — a compact India-first booster (+ one hard gate)

Frontier LLMs are weak here (MapEval: best ~67% vs 86.7% human) and no dedicated Indian geo-LLM exists — an unclaimed lane. Added as a small cross-cutting rider, not a new pillar; cartographic correctness is the exception — it gates release.

Cartographic sovereignty INDIA-FIRST — default to Survey-of-India boundaries (J&K, Ladakh/Aksai Chin, PoK, Arunachal as Indian). Wrong depiction is criminal in India (up to 7 yrs / ₹100 cr); Western models render Kashmir "disputed" by viewer geography, so a US-trained model is non-compliant by default. Hard gate: ~150–300-item boundary probe, target ~100%, launch-blocking.
Admin depth — the LGD hierarchy (28+8 states/UTs, 785 districts, ~6,471 sub-districts, ~584,615 villages) + PIN codes → address parsing, civic/agentic tasks, disaster response.
How much (compact)
  • Pre-train: ~40B-token geo slice (~0.33% of 12T, ~half India) — GeoLLM-style OSM textualization + LGD/PIN/Census serialized to text + explicit boundary statements. Don't oversample (it displaces code/agentic).
  • SFT: ~60–100k — geo tool-use AGENTIC + spatial-reasoning CoT + India-map QA.
Precedent (text, scale-appropriate): GeoGalactica (30B, geo continued-pretrain), GeoLLM (OSM→text, improves with scale), K2. EO/vision models (Prithvi, Clay, GeoChat) are complementary — orchestrate via tools, don't replicate.

8Top risks

  • Indic native scarcity (~120–150B) → synthetic slice risks translationese; track synthetic:real.
  • Indic coding & agentic evals don't exist publicly → build in-house (a differentiator).
  • BNS/BNSS/BSA + current-affairs evals go stale in months → refresh on a cadence.
  • Legal clearance — Stack v2 opt-outs; copyright on judgments & news.
  • Cartographic gate is legally load-bearing — boundary-depiction errors are criminal in India; treat regressions as launch blockers.
  • Dravidian fertility floor ~2–2.5 even at 200K; freeze targets after measuring on our own corpus.
